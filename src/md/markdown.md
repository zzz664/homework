# 마크다운 문법

### 줄바꿈

줄 바꿈은 \<br>을 이용하거나 띄어쓰기를 두번 사용하여 할 수 있음

### Header

"#" 기호의 개수를 조합하여 h1~6 글머리를 표현할 수 있다.

# 이건 h1 글머리입니다.

    # 이건 h1 글머리입니다.

## 이건 h2 글머리입니다.

    ## 이건 h2 글머리입니다.

### 이건 h3 글머리입니다.

    ### 이건 h3 글머리입니다.

#### 이건 h4 글머리입니다.

    #### 이건 h4 글머리입니다.

##### 이건 h5 글머리입니다.

    ##### 이건 h5 글머리입니다.

###### 이건 h6 글머리입니다.

    ###### 이건 h6 글머리입니다.

### BlockQuote

">" 기호를 이용하여 인용문을 표현 할 수 있다. 인용문 안에는 다른 마크다운 요소를 작성 할 수 있음

> 인용문 입니다.
>
> > ### 인용문 안쪽에 h3요소를 작성했습니다.
> >
> > > 다중 중첩 인용문입니다.

```
> 인용문 입니다.
>> ### 인용문 안쪽에 h3요소를 작성했습니다.
>>> 다중 중첩 인용문입니다.
```

### Orderd List(순서있는 목록)

"숫자 + 점"의 형태로 순서있는 목록을 작성 할 수 있다. 아무 숫자나 입력해도 내림차순 정렬되어 나옴

1. 첫번째
2. 두번째
3. 세번째

```
1. 첫번째
2. 두번째
3. 세번째
```

### Unorderd List(순서없는 목록)

"\*, +, -" 기호를 이용하여 순서없는 목록을 작성 할 수 있다.

- 첫번째
  - 두번째
    - 세번째

```
* 첫번째
    * 두번째
        * 세번째
```

- 첫번째
  - 두번째
    - 세번째

```
+ 첫번째
    + 두번째
        + 세번째
```

- 첫번째
  - 두번째
    - 세번째

```
- 첫번째
    - 두번째
        - 세번째
```

세 가지 기호를 혼합하여 사용 할 수 있음

- 첫번째
  - 두번째
    - 세번째
      - 네번째

```
* 첫번째
    - 두번째
        * 세번째
            + 네번째
```

### Code

들여쓰기(4개의 공백이나 탭)를 만나면 들여쓰지 않은 행을 만날때까지 Code Block으로 변환된다.  
들여쓰기 한 행과 들여쓰기 하지 않은 행은 한줄을 띄어써야 정상 작동한다.

들여쓰지 않은 행입니다.

    들여쓰기 한 행입니다.
    들여쓰기 한 행입니다.
    들여쓰기 한 행입니다.

들여쓰지 않은 행입니다.

```
들여쓰지 않은 행입니다.

    들여쓰기 한 행입니다.
    들여쓰기 한 행입니다.
    들여쓰기 한 행입니다.

들여쓰지 않은 행입니다.
```

### Link

- 참조링크  
  [Naver][naverlink]  
  [naverlink]: https://naver.com "Go to Naver"

```
[Naver][naverlink]
[naverlink]: https://naver.com "Go to Naver"
```

- 외부링크  
  [Naver](https://naver.com, 'naver link')

```
[Naver](https://naver.com, "naver link")
```

- 자동연결  
  외부링크: <https://example.link/>  
  이메일링크: <example@email.com>

```
외부링크: <https://example.link/>
이메일링크: <example@email.com>
```

### Table

테이블 헤더를 구분하기 위해 3개이상의 `-`를 사용한다.  
`:`를 이용하여 셀 내용을 정렬할 수 있다.  
`---`, `:---` : 좌측 정렬  
`:---:` : 가운데 정렬  
`---:` : 우측 정렬  
| 값(좌측정렬) | 의미(가운데정렬) | 기본값(우측정렬) |
| :--- | :---: | ---: |
| `static` | 유형(기준)없음 / 배치불가능 | `static` |
| `relative` | 요소 자신을 기준으로 배치 | 없음 |
| `absolute` | 부모 요소를 기준으로 배치 | 없음 |
| `fixed` | 브라우저 창 기준으로 배치 | 없음 |
| `sticky` | 스크롤 영역 기준으로 배치 | 없음 |

이때 가장 좌측, 우측의 `|`는 생략이 가능하다.
